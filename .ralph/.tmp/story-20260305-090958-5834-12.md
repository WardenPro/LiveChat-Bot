### US-012: Refactor media lifecycle internals for correctness and cleanup safety
Status: in_progress
Depends on: US-006, US-007

Description:
As an operator, I want media ingestion and cleanup internals simplified and guarded so that temporary files and asset states remain correct under failures.

Acceptance Criteria:
- [ ] Split large media lifecycle logic into focused modules/functions with explicit typed boundaries.
- [ ] Preserve existing storage paths, media URL behavior, and expiration semantics.
- [ ] Example: successful media ingest produces the same playable asset behavior and metadata state transitions as baseline.
- [ ] Negative case: failed download/transcode path cleans temporary resources and does not leave inconsistent asset status beyond baseline behavior.
- [ ] Characterization checks cover success, timeout/error, and cleanup scenarios.
