### US-009: Tighten TypeScript compiler policy with tracked exceptions
Status: in_progress
Depends on: US-001

Description:
As a maintainer, I want stricter TypeScript settings and explicit exception tracking so that runtime bugs are prevented earlier.

Acceptance Criteria:
- [ ] Adopt stricter tsconfig policy for refactor target (including noImplicitAny and additional strictness flags) using a phased, non-breaking rollout.
- [ ] Where full strictness cannot be immediate, add temporary localized exceptions with TODO owner and removal ticket reference.
- [ ] Example: touched modules compile without implicit any and with explicit interface/type contracts.
- [ ] Negative case: introducing a new implicit any in a touched path fails build/lint checks.
- [ ] Document final strictness matrix and remaining exception backlog.
