### US-004: Refactor socket loader lifecycle with strict boundaries
Status: in_progress
Depends on: US-002

Description:
As a maintainer, I want socket authentication, session lifecycle, and event emission separated into explicit modules so that correctness and security are easier to enforce.

Acceptance Criteria:
- [ ] Split socket loader internals into authentication, connection state, and event dispatch modules with explicit typed interfaces.
- [ ] Preserve event names and payload contract from @livechat/overlay-protocol.
- [ ] Example: authenticated overlay client receives the same event sequence and payload structure as baseline.
- [ ] Negative case: invalid token or malformed payload follows the same rejection/disconnect semantics as baseline.
- [ ] No new global mutable state is introduced outside existing runtime wiring constraints.
