### US-001: Establish characterization baselines for current behavior
Status: in_progress
Depends on: None

Description:
As a maintainer, I want executable characterization coverage for current contracts so that refactoring can proceed without accidental behavior drift.

Acceptance Criteria:
- [ ] Create characterization scripts/tests for REST, Socket.IO lifecycle, Discord command execution flow, and media lifecycle critical paths in touched modules.
- [ ] Baseline artifacts capture request/response status codes, payload shapes, and key side effects for sensitive routes and events.
- [ ] Example: valid POST /overlay/pair/consume returns the same status code and response shape in baseline and post-change runs.
- [ ] Negative case: malformed consume payload reproduces the same validation error code/body shape as baseline.
- [ ] A runbook documents how to execute characterization checks before each refactor slice.
