### US-006: Centralize input validation and request parsing
Status: in_progress
Depends on: US-003, US-004, US-005

Description:
As a maintainer, I want shared validation and parsing utilities so that inputs are consistently sanitized and typed across domains.

Acceptance Criteria:
- [ ] Introduce shared validation helpers (using existing validation approach) for common request body/params/query patterns.
- [ ] Apply shared validators to sensitive REST and socket input paths without changing success/error contract semantics.
- [ ] Example: valid admin payload (including expected author identifiers) is accepted and persisted exactly as before.
- [ ] Negative case: invalid authorDiscordUserId still yields baseline-compatible invalid_author_discord_user_id behavior.
- [ ] All new validators expose explicit TypeScript input/output types.
