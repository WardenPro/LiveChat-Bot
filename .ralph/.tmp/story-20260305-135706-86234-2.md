### US-002: Create module inventory and missing-test gate
Status: in_progress
Depends on: US-001

Description:
As a maintainer, I want an automated runtime module inventory so that we can enforce one-or-more unit tests for every module.

Acceptance Criteria:
- [ ] Add a deterministic inventory script that scans src runtime modules and excludes characterization, architecture docs, declaration files, and typecheck-only files.
- [ ] Add script pnpm test:unit:matrix that fails when inventory finds modules with no mapped test files.
- [ ] Example: when a module has a matching test file, test:unit:matrix reports it as covered.
- [ ] Negative case: when a new module is added without a test file, test:unit:matrix fails and prints the missing module path.
- [ ] Matrix output is machine-readable enough for CI logs and developer troubleshooting.
