### US-011: Perform dependency hygiene with patch-only updates
Status: in_progress
Depends on: US-001

Description:
As a maintainer, I want safe dependency hygiene so that known risks are reduced without introducing compatibility breaks.

Acceptance Criteria:
- [ ] Audit direct dependencies for unused packages, duplicated transitive risks, and available patch updates.
- [ ] Apply only patch-level upgrades with release note review and rollback notes.
- [ ] Example: a patch security update is applied and characterization/lint/build checks remain green with no contract changes.
- [ ] Negative case: minor or major update candidates are explicitly deferred and documented as out of current scope.
- [ ] Update dependency hygiene report with before/after versions and rationale.
