### US-005: Refactor Discord command loading and execution flow
Status: in_progress
Depends on: US-002

Description:
As a maintainer, I want Discord command registration and execution internals modularized so that command maintenance is safer and clearer without user-visible changes.

Acceptance Criteria:
- [ ] Split Discord loader responsibilities into command registry, command metadata assembly, and interaction execution handling.
- [ ] Keep command names, options, permission expectations, and user-facing responses unchanged.
- [ ] Example: /help command returns the same embed/message content and status behavior as baseline.
- [ ] Negative case: unknown or failing command interaction is handled safely without process crash and with baseline-equivalent user-facing outcome.
- [ ] Characterization checks confirm no contract drift in command registration payload shape.
