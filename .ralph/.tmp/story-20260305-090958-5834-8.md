### US-008: Harden environment parsing and secure defaults
Status: in_progress
Depends on: US-006

Description:
As a maintainer, I want stricter runtime config parsing and secure defaults so that invalid states are reduced without altering valid deployment behavior.

Acceptance Criteria:
- [ ] Refactor env parsing into clearly typed config modules with explicit defaults and invariant checks.
- [ ] Preserve behavior for all currently valid environment configurations.
- [ ] Example: existing production-equivalent env values produce identical runtime configuration outputs.
- [ ] Negative case: clearly invalid config value is rejected deterministically with safe logging and controlled startup failure behavior.
- [ ] No environment variable is renamed or removed; backward-compatible aliases are documented if needed.
