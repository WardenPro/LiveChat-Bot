### US-005: Backfill tests for media service pipeline
Status: in_progress
Depends on: US-001, US-002, US-004

Description:
As a maintainer, I want unit tests for media ingestion, caching, source resolving, and lifecycle orchestration so that transient media behavior stays reliable.

Acceptance Criteria:
- [ ] Cover src/services/media/* modules and related error types, including ingestion, transcode boundaries, cache interactions, and cleanup logic.
- [ ] Use filesystem/temp-directory mocks to keep tests deterministic and side-effect safe.
- [ ] Example: valid transient media input stores expected metadata and returns a resolvable transientMediaId.
- [ ] Negative case: unsupported media format or failed source resolution returns typed media error and triggers cleanup path.
- [ ] All media modules are listed as covered by pnpm test:unit:matrix.
